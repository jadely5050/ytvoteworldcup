# 치지직(CHZZK) 연동 — 위험성 및 현재 상황 정리

> 작성일: 2026-06-20 · 대상: ytvoteworldcup 라이브 투표 오버레이
> 한 줄 요약: **치지직 연동은 구현 완료되어 main에 머지된 상태이며, 비공식 라이브러리 기반이지만 위험도는 유튜브와 비슷한 수준(낮음~중간)이다.**

### 작동 방식
1. admin에서 "치지직 채팅 수집" 체크 + 채널 ID 입력 후 저장
2. 서버가 `ChzzkClient`로 해당 채널의 **라이브 상태(chatChannelId·accessToken)를 익명으로 조회**
3. 네이버 채팅 서버(`wss://kr-ssN.chat.naver.com`)에 WebSocket 연결
4. 들어온 채팅을 유튜브와 **동일한 집계 로직**(`ingestChat` → `countChatVote`)으로 처리
5. 채널 ID에 `chzzk:` 접두사를 붙여 플랫폼 간 중복투표 방지

## 2. 위험성 분석 (Risk Assessment)

### (가) 비공식 API — 라이브러리 붕괴 위험 · 중간
- 네이버 공식 실시간 수신 API가 아니라, 치지직 내부 통신을 **리버스 엔지니어링한 비공식 라이브러리**다.
- 네이버가 채팅 서버 프로토콜·토큰 방식을 바꾸면 **라이브러리가 갑자기 먹통**이 될 수 있다.
- 다만 유튜브용 `masterchat`도 똑같은 성격이고, 치지직 비공식 라이브러리는 커뮤니티가 활발해 **패치가 비교적 빠른 편**이다.
- **영향**: 패치 전까지 치지직 수집만 중단. 유튜브 투표는 영향 없음(소스 독립 구조).

### (나) 계정 밴 위험 · 매우 낮음
- **로그인 없이 읽기만** 하므로 "밴당할 내 계정" 자체가 없다.
- 채팅 전송 등 인증 기능을 쓰지 않는 한 계정 제재 시나리오는 사실상 없음.

### (다) 차단(IP/연결) 위험 · 낮음
- 이론상 네이버가 비정상 트래픽으로 판단해 연결을 끊을 수 있으나, 공개 라이브 채팅 단순 수신은 부담이 적어 **실제 차단 사례는 드물다.**
- 차단되더라도 자동 재연결(15초 디바운스) 로직이 있어 일시적 끊김은 스스로 복구한다.

### (라) 공식 API 등장 → 비공식 라이브러리 deprecate 가능성 · 낮음~중간
- 네이버가 **공식 Open API**(`openapi.chzzk.naver.com`, client_id/secret OAuth)를 출시했다.
- 단, 현재 공식 API는 **채팅 송신·공지·방송 설정** 위주이고, 우리가 쓰는 **실시간 채팅 "수신"은 여전히 비공식 라이브러리가 표준**이다.
- 일부 비공식 라이브러리는 "공식 API 정식 발표 시 업데이트 중단"을 명시 → 장기적으로는 공식 수신 API로 갈아탈 필요가 생길 수 있다.

### (마) 운영상 흔한 "안 되는" 상황 (위험이라기보단 사용 주의)
- **방송이 꺼져 있으면** 채팅이 안 들어온다. 치지직 채팅은 **해당 채널이 실제 라이브 중일 때만** 붙는다(오프라인이면 15초마다 재연결 시도).
- **채팅은 오는데 투표 카운트가 안 되는 경우** → 집계 규칙 문제이지 치지직 문제가 아니다:
  - "집계 키워드 사용" 토글이 꺼져 있으면 키워드 무시(별표 `*1`만 집계)
  - 한 글자 키워드('ㅋ')는 메시지가 딱 그 글자일 때만 인정 → `ㅋㅋ`처럼 2글자+ 권장

## 3. 플랫폼 비교

| | 유튜브 | **치지직** | 틱톡 |
|---|---|---|---|
| 방식 | 비공식(masterchat) | **비공식(chzzk)** | 비공식 + 외부 사인서버 |
| 인증/키 | 영상 ID만 | **채널 ID만** | 사인서버 의존 |
| 계정 밴 위험 | 없음(익명) | **없음(익명)** | 없음(읽기만 시) |
| 갑자기 끊길 위험 | 낮음 | **낮음~중간** | 높음 |
| 외부 유료 의존 | 없음 | **없음** | 있을 수 있음 |
| 한글 투표 적합성 | 높음 | **매우 높음(국내)** | 보통 |
| 종합 위험도 | 낮음 | **낮음~중간** | 중간~높음 |

→ **치지직은 유튜브와 거의 동급**으로, 국내 시청자 투표에는 오히려 가장 적합하다.

---

## 4. 운영 권장 사항

1. **소스 독립 구조 유지** — 치지직이 끊겨도 유튜브 투표는 정상 동작(현재 구조가 이미 이러함). 한 플랫폼 장애가 전체로 번지지 않음.
2. **방송 시작 후 연결 확인** — 서버 콘솔의 `[chzzk] 채팅 수신 시작 ✔` 로그로 확인. `연결 실패`가 반복되면 방송이 꺼졌거나 채널 ID 오류.
3. **중요 생방송 전 리허설** — 실제 채널로 사전 테스트(채팅 표출 + `*1` 투표 반영)를 권장.
4. **라이브러리 버전 고정 & 주기적 업데이트** — `package.json`에 버전 고정, 방송 전 `chzzk` 최신 패치 확인.
5. **장기 대비** — 네이버 공식 수신 API가 정식화되면 `sources/chzzk.js`를 공식 API 기반으로 교체(현재 어댑터 구조라 교체 용이).

---

## 5. 결론

- 치지직 연동은 **이미 동작하며 위험도는 낮은 편**이다. "계정이 밴된다"류의 위험은 사실상 없고, 가장 현실적인 리스크는 **비공식 라이브러리가 네이버 변경으로 일시 중단될 가능성**이다.
- 그마저도 **소스 독립 구조**라 치지직만 영향을 받고 유튜브는 멀쩡하다.
- **국내 라이브 투표 용도로는 가장 안전하고 적합한 추가 플랫폼**이라고 평가한다.
