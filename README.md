# ytworldcup — 라이브 채팅 투표 → vMix 오버레이

**YouTube · 치지직(CHZZK)** 라이브 채팅을 받아 **실시간 투표를 집계**하고,
투명 배경 HTML5 오버레이로 그려 **vMix의 Web Browser 입력**으로 띄우는 로컬 서버입니다.
별도 캡처카드·유료 API 없이 동작하며, 모든 설정은 웹 **admin 페이지**에서 합니다.

## 주요 기능

- **멀티 플랫폼 채팅 수집** — YouTube + 치지직을 각각 on/off, **동시 집계** 가능
- **여러 투표 방식** — 별표 투표(`*1`) · 키워드 매칭 · 유튜브 네이티브 설문
- **그래픽 2종** — 막대 그래프 / 그림(스틸·이미지 시퀀스)
- **득표 연출** — 그림 들썩임("띠용") **강도 보기별 1~5 조절**
- **채팅창 표시 정책** — 투표한 채팅만 / 모두 보이기
- **운영 도구** — 송출(ON-AIR) 토글, 집계 시작/종료, 프리셋 저장·불러오기, 테스트 시뮬레이터
- **공식 API·키 불필요** — 비공식 크롤링 라이브러리 사용(쿼터 무제한)

## 동작 방식

```
 YouTube 라이브 ──(masterchat 크롤링)──┐
                                       ├──▶ server.js
 치지직 라이브  ──(chzzk WebSocket)────┘      · 별표/키워드 집계, 네이티브 설문 감지
                                              · 채팅 표시 정책 필터
                                              │  WebSocket 실시간 푸시
                                              ▼
                                   public/overlay.html (투명 배경)
                                              │
                          vMix ▶ Add Input ▶ Web Browser ▶ localhost:3000/overlay.html
```

## 설치 & 실행

```powershell
npm install
npm start
```

콘솔에 오버레이 URL과 활성 채팅 소스가 출력됩니다. 브라우저로 `http://localhost:3000/` 에 접속하면 **admin 페이지**가 열립니다. (상세 설치는 `install.md` 참고)

## 설정 (admin 페이지)

`config.json`을 직접 편집할 필요 없이 admin 페이지에서 모두 설정하고 **저장**하면 즉시 반영·재연결됩니다. 좌측 폼 / 우측 미리보기(PGM·PVW) 2단 구성입니다.

### 1) 채팅 소스
- **유튜브 채팅 수집** 체크 + 라이브 주소/영상 ID (URL 붙여넣으면 ID 자동 추출)
- **치지직 채팅 수집** 체크 + 채널 ID (`chzzk.naver.com/live/...` 주소도 가능)
- 둘 다 켜면 **두 플랫폼 채팅이 같은 투표에 합산**됩니다. 각 소스는 독립적으로 재연결되어 한쪽이 끊겨도 다른 쪽에 영향 없음.

### 2) 투표 방식 (poll.mode)
| mode | 설명 |
|------|------|
| `keyword` | 채팅 키워드/별표 입력을 집계 |
| `native`  | 유튜브 네이티브 설문 결과만 표시 |
| `auto`    | 네이티브 설문이 감지되면 우선, 없으면 키워드 집계 |

- **별표 투표**: 채팅이 `*1`·`*2`처럼 별표+숫자 **단독**이면 보기 1·2번으로 집계. "집계 키워드 사용" 여부와 무관하게 **항상** 인정 → 가장 확실.
- **키워드 투표**: "집계 키워드 사용"을 켜야 동작. 보기별 등록 키워드로 집계. **부분일치는 2글자 이상**일 때 적용(`ㅋㅋ`는 `ㅋㅋㅋ`에도 매칭, 한 글자 `ㅋ`은 단독일 때만).
- **한 사람당 1표** 옵션(체크 해제 시 중복 투표 누적).

### 3) 그래픽 / 그림 모드
- **막대 그래프** 또는 **그림(PNG)** 모드 선택.
- 그림은 보기마다 업로드: **1장=스틸, 여러 장=이미지 시퀀스**(애니메이션, fps 조절).
- **띠용 강도(1~5)**: 보기 카드의 그림 영역 우측 슬라이더. 득표 시 그림이 들썩이는 강도를 조절(잔잔 1 — 5 방방, 기본 3).

### 4) 채팅창 표시
- **우측 채팅창 표시** on/off.
- **보여줄 채팅**: `투표한 채팅만 보이기`(기본) / `모두 보이기`. 투표 집계는 두 경우 모두 동일하게 동작하고, 채팅창 노출 범위만 제어합니다.

### 5) 운영 도구
- **ON-AIR**: 오버레이 그래픽 fade in/out 송출 토글.
- **집계 시작/종료**: 종료 시 득표를 동결(현재 결과 유지).
- **프리셋**: 투표 이름으로 저장/불러오기 (`presets/*.json`).
- **테스트 시뮬레이터**(`/test.html`): 유튜브·치지직 없이 채팅을 주입해 집계/연출 점검.

## vMix 설정

1. **Add Input → Web Browser**
2. URL: `http://localhost:3000/overlay.html`
3. Width `1920`, Height `1080`
4. **Transparent Background 체크** ← 투명 합성 핵심
5. 이 입력을 Overlay 채널에 올려 라이브 영상 위에 합성

> 같은 PC면 위 설정으로 끝. 렌더 PC와 vMix PC가 분리된 경우에만 렌더 PC에서 `overlay.html`을 풀스크린으로 띄워 SDI 카드로 캡처해 입력하세요.

## 주의

- **비공식 라이브러리 기반** — YouTube는 `masterchat`, 치지직은 `chzzk` 비공식 라이브러리를 씁니다. 공식 실시간 댓글 API가 없어 채택한 방식으로, 플랫폼 내부 포맷이 바뀌면 깨질 수 있습니다(`npm update` 로 패치). **로그인 없이 공개 채팅 읽기만** 하므로 계정 위험은 없습니다.
- **치지직은 방송 중일 때만** 채팅이 들어옵니다(오프라인이면 자동 재연결 대기).
- **틱톡은 미지원** — 기술적으로는 가능하나 외부 사인서버 의존·차단·약관 리스크로 제외했습니다. (검토 내역은 `docs/` 위험성 문서 참고)
- 공식 YouTube Data API는 쿼터 한계로 장시간 라이브에 부적합해 크롤링 방식을 채택했습니다.
